# Contributing

This repository is primarily a portfolio artifact, but contributions and review
comments are welcome if they improve clarity, correctness, or commercial
usefulness.

## Before opening a pull request

1. Keep examples generic, synthetic, and non-client.
2. Do not add internal workplace material or real operating-model documents.
3. Run:

   ```bash
   make test
   ```

4. Update README or docs when architecture scope, templates, or diagrams change.
5. Explain the business reason for the change, not only the wording edit.

## Review standard

A change is not ready if it adds governance language without making the playbook
easier to use, review, or adapt.
